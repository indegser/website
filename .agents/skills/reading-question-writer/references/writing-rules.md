# Reading Question Writing Rules

A reading question post should turn a specific confusion into a reusable explanation.

## Shape

Use this order unless the user asks for a different one:

1. Direct answer
2. Why the confusion happens
3. Explanation
4. Concrete example
5. Key terms
6. What became clear
7. Open questions

## Title

Prefer concept-first titles.

Good:

- `USDC가 그냥 달러 보유와 다른 점`
- `양자컴퓨팅이 블록체인 보안에 주는 실제 위협`

Avoid:

- `질문 정리`
- `USDC 질문`
- `내가 궁금했던 것`

## Prose

- Write in natural Korean unless the user asks otherwise.
- Answer the question early.
- Keep definitions short.
- Use examples when the concept is abstract.
- Do not overstate certainty on finance, law, health, or fast-changing topics.

## Blocks

- Use a `callout` block with `tone: "question"` for the original question when it is useful to show it.
- Use normal Portable Text headings for sections.
- Use bullet lists only when scannability improves.
